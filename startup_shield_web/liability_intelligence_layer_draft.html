liability intelligence layer

Read the risk. Explain the product. Route the action.

This layer sits between the IIB workbook and the dashboard. It normalizes messy raw fields, retrieves the right liability knowledge, explains the product in plain English, and returns separate recommendation paths for RMs and Underwriters.

Input IIB sheet schema

Raw policy, claims, and dictionary fields. Missing values allowed.

Core output Decision objects

RM next-best-action and UW referral logic from one normalized record.

Future proof Multi-line ready

Fire, engineering, health, and bundles plug into the same adapter model.

Layer architecture

The layer behaves like a controlled pipeline, not a chatbot. Every stage produces structured output, confidence, and explanation text.

1. ingest Workbook + doc + future feeds Policy rows, claims rows, dictionary schema, product notes.
→
2. normalize Canonical risk object Product family, exposures, missing fields, data quality.
→
3. retrieve Relevant product text Clauses, exclusions, cover intent, and product logic.
→
4. decide RM and UW outputs Pitch, refer, price, or ask for more data.

What it returns

rm output

Attack order

Which product to lead with, why it matters, and what adjacent cover to pitch next.

uw output

shared output

Explanation

A single defensible reason chain that both sales and underwriting can trust.

Draft intent: a decision layer, not a prettier report
